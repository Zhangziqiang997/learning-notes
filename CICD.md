# 是什么

1. **持续集成 (Continuous Integration - CI)**：开发人员频繁地将代码合并到主干分支。每次合并后，系统都会**自动构建**并运行**自动化测试**。
2. **持续交付 (Continuous Delivery - CD)**：在 CI 的基础上，将通过所有测试的代码自动打包成一个可以随时部署的“发布包”（在我们的场景里，就是一个 Docker 镜像），并可能自动部署到“预发布”或“测试”环境。
3. **持续部署 (Continuous Deployment - CD)**：这是最激进的模式。在持续交付的基础上，连最后“按按钮”的步骤都自动化了。任何通过所有自动化测试的代码变更，都会被自动地、直接地部署到生产环境。

**好处**：加快交付速度、降低发布风险、提高代码质量、让开发人员更专注于业务逻辑

# CI步骤
1. **自动构建**    
    - **做什么：** 将源代码编译、打包，生成可以运行的“制品”。例如：将Java代码编译成Jar/War包，或将应用及其环境打包成一个Docker镜像。        
    - **工具举例：** `Maven`、`Gradle`、`npm`、`Docker`。        
    - **面试点：** 构建的产出物是“制品”，它是后续所有环节的起点，必须被妥善存储（如存放在Nexus, Jfrog Artifactory, Docker Registry中）。        

2. **自动测试**    
    - **做什么：** 运行一系列自动化测试来验证代码质量。        
    - **测试金字塔：**        
        - **单元测试：** 底层，数量最多，测试单个函数/方法。快速、隔离。（工具：JUnit, TestNG, Jest）      
        - **集成测试：** 中层，测试多个模块间的协作，例如Service与Database的交互。            
        - **端到端测试：** 顶层，数量少，从用户界面模拟真实用户操作，验证整个系统。速度慢，但信心度高。（工具：Selenium, Cypress）            
    - **面试点：** 理解测试金字塔，知道如何平衡测试的广度、速度和成本。

3. **自动反馈**    
    - **做什么：** 将构建和测试的结果（成功/失败）即时通知给开发团队。如果失败，则阻塞本次变更合入主干，团队优先修复。

**CI的目标：** 快速发现错误，防止分支大幅偏离主干，保证软件仓库始终处于健康状态

# CD

持续交付与持续部署：解决“发布”的焦虑
CI保证了代码质量，CD则负责将高质量的代码安全地送出去。
1. **制品管理：** 将CI阶段产生的、经过验证的可靠制品（如`my-app:v1.2.3`镜像）上传到制品仓库，并做好版本标记。    
2. **自动化部署与发布策略：**    
    - **做什么：** 将新版本的制品部署到各类环境（测试、预生产、生产）。        
    - **部署策略（高级面试重点）：**        
        - **蓝绿部署：** 准备两套完全相同的环境（蓝和绿）。当前流量在“蓝”环境（v1）。将v2部署到“绿”环境，测试通过后，直接将流量从“蓝”切换到“绿”。优点是秒级回滚（切回蓝即可）。            
        - **金丝雀发布：** 将新版本v2先部署给一小部分用户（比如1%的服务器或用户），验证没问题后，再逐步扩大范围至全量。这是一种风险较低的发布方式。
3. **自动化后置流程：**    
    - **自动化测试（生产环境）：** 部署后自动运行一些健康检查（如`/health`端点）和冒烟测试。        
    - **监控与反馈：** 密切监控应用性能指标（APM）、日志和业务数据，形成闭环反馈。

# 核心工具链
|环节|代表性工具|
|---|---|
|**CI平台（管道引擎）**|**Jenkins**（自建，灵活强大）， **GitLab CI**（与GitLab深度集成）， **GitHub Actions**（与GitHub深度集成）|
|**代码仓库**|**Git**（GitLab, GitHub, Bitbucket）|
|**构建与打包**|**Maven/Gradle**（Java）， **npm/yarn**（JS）， **Docker**（容器化）|
|**制品仓库**|**Nexus**， **Jfrog Artifactory**， **Docker Registry**（Harbor）|
|**部署与编排**|**Ansible**（自动化运维）， **Kubernetes**（容器编排）， **Terraform**（基础设施即代码）|
|**监控**|**Prometheus**（监控）， **Grafana**（可视化）， **ELK**（日志）|
